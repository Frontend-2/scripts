# Script clase 1
## Bienvenida e introduccion a JavaScript front2

### Introducción a la materia - Conocer el origen de JS - Algunos usos de la consola y vinculación HTML Y JS	
## Clase en vivo

### 1) Bienvenida - 5 min
###### Descripción: Espera a que llegue todo el grupo, pone un poco de música para esperar y va saludando a quienes ingresan.
Tarea de todas las clases por parte del tutor/a:
- Tomar asistencia con egg
- Armar las mesas de trabajo en egg
- Preparar las salas 
- Activar las salas cuando sea necesario"

### 2) Inicio - 10 min
- El profesor releva las dudas que hayan surgido en el form.

### 3) Explicación de un uso 5 min
- Momento de presentar un desafío. En este caso utilizaremos un caso de usuario para analizar y poder realizar una solución. Lo que vamos a tener que resolver es la realización de un juego de piedra, papel o tijera con las herramientas que tenemos hasta el momento.

- La idea es que el profesor pueda abordar este caso de usuario y vaya recorriendo los requerimientos planteados de la mano de los alumnos. El profesor, entonces, puede empezar a consultarles a los alumnos que herramientas del lenguaej posiblemente estén involucradas en esta solución: condicionales, alertas, etc.

- A continuación, el profesor empieza a plantear una solución en Visual Studio Code.

- [Link complementario] https://drive.google.com/file/d/1ZnZluxnZcGuskLr8HOYOjdbjf9BY7UE_/view?usp=sharing

### 4) Demostración de código - 20 min
- El profesor codea la solucion frente a los alumnos haciendo que participen del proceso indagando en como continuar el código: ""¿Que podemos hacer aquí?¿cómo podemos continuar con este procedimiento?

- La idea de este momento es que el profe termine de resolver la problemática con las herramientas que los alumnos tienen hasta el momento: muchos condicionales, variables, y todo el código en un único bloque..

- [Link complementario] https://docs.google.com/document/d/1wOBDGkGhWlKrFRyW2lGOV_nJuneUR3Gsa0hiRHX39nk/edit?usp=sharing

### 5) Code review - 10 min
El profesor realiza un repaso del codigo realizado y aquí empieza a presentar la consigna para el espacio de mesas de trabajo: refactorizar el código.
La idea es que, con la solución planteada, los alumnos puedan ""mejorar"" el código. ¿Cuáles son algunas de las mejoras que estamos buscando? Enumeramos algunas:

- Que utilicen otro tipo de condicional (switch por ejemplo)
- Que puedan separar todo este código en funciones mas chiquitas (Este punto quizás no lo tienen muy presente, así que se podría resolver al menos una mini función que genere un número random)

¿Qué preguntas disparadoras puede hacer cuando lea la consigna? Vamos con algunos ejemplos:
- ¿Qué se les ocurre en esta parte?
- ¿Qué herramientas tenemos para poder recorrer un array?
- ¿Podríamos usar X cosa para resolver esta parte?

Una palabra clave de este momento es ITERACIONES: algo que ya deberían tener visto de programación imperativa"

### 5) Cierre de clase - 10 min
Se presenta de manera formal el enunciado de la práctica para el espacio de mesas y se genera un breve cierre
