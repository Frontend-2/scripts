# Script clase 18
## Asincronismo y APIs
### OBJETIVOS: Agregar el metodo POST. AJAX, Fetch, Postman

## Clase en vivo
​
### 1) Bienvenida - 5 min
###### Descripción:  Espera a que llegue todo el grupo, pone un poco de música para esperar y va saludando a quienes ingresan.
Tarea de todas las clases por parte del tutor/a:
- Tomar asistencia con egg
- Armar las mesas de trabajo en egg
- Preparar las salas 
- Activar las salas cuando sea necesario"
​
### 2) Debate - 10 min
- Se charla en conjunto los puntos de APIs trabajados hasta el momento, la idea es repasar lo que hicimos con Fetch GET para profundizar con Fetch POST.
​
### 3) Explicación - 10 min
Para diferenciar acerca de la manera en que trabajamos el GET y el POST, el profe puede valerse de 2 slides que resumen muy brevemente lo que los estudiantes deben considerar al trabajar con las distintas peticiones.
Imagen azul ->  GET
Imagen verde -> POST
GET: - [imagen](https://drive.google.com/file/d/1Eq8z_KJy7QofCDBhoV8dSRFcyxLuBMq7/view?usp=sharing)
POST: -[imagen](https://drive.google.com/file/d/18Zo_3KkcPZ_9dNOVVCJnnb39KXIfk6iI/view?usp=sharing)
​
### 4)Postman + JSON placeholder - 20 min
Para comenzar con algunas practicas de POST con la heramienta Postman, el profe puede implementar la API de JSON placeholder para realizar distintas pruebas que no generan problema en niguna base de datos ya que son respuesta simuladas.
[Link complementario](https://jsonplaceholder.typicode.com/guide/)
Ejemplo de POST + dataset: - [Link complementario](https://drive.google.com/file/d/1nFxerOBprr8wdFkZt4sy1IFiSt7JoPnH/view?usp=sharing) 
​
### VS CODE - 15 min
###### Finalmente, antes de enviarlos a las mesas de trabajo se pueden realizar la creación de un usuario en la API de To-Do para que así los estudiantes vean como funciona y logren crearse sus usuarios en la API.
​[Heroku app](https://ctd-todo-api.herokuapp.com/)

### EJERCITACION - 50 min TUTOR
- Si bien pueden ir a mesas, los estudiantes individualmente deberan lograr obtener el token a traves de la creacion de un usuario en la API de To-Do.
​[Link a Ejercitacion](https://docs.google.com/document/d/1ZiCPf7IICvtp6rwfxoq5Wh5dJUROKqNw/edit).
